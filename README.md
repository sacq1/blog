# 岩行记 — 户外攀岩个人博客

一个轻量级、纯静态的个人博客网站，专为户外运动爱好者设计。部署在 **GitHub Pages** 上，零成本、零后端、零数据库，所有内容通过浏览器 localStorage 管理。

## 网站做什么的？

这是**岩行记**的个人博客网站，记录户外攀岩、徒步旅行等内容，包含以下模块：

| 页面 | 功能 |
|------|------|
| **首页** | 网站入口，展示最新3篇文章、岩行中国预告、作者简介 |
| **文章页** | 所有博客文章列表，支持按分类筛选（攀岩、徒步、攻略等） |
| **技术分享** | 攀岩技术资料库，支持上传 PDF 文件，访客可一键下载 |
| **岩行中国** | 中国攀岩场地探索路线图，记录每个目的地的攻略 |
| **关于页** | 作者介绍、联系方式和隐藏的博主管理入口 |

## 特色功能

- **深色/浅色主题切换** — 一键切换，自动记忆偏好
- **富文本文章编辑** — 支持 Markdown 语法、图片插入
- **文章显隐管理** — 博主可随时隐藏/发布文章
- **PDF 技术资料分享** — 上传 PDF 文件供访客下载，文件存储在浏览器本地
- **权限控制** — 密码保护的博主后台，只有作者可以编辑和上传
- **移动端适配** — 响应式设计，手机和平板都能正常浏览
- **自动刷新** — 网站内容更新后自动检测并刷新

## 技术栈

纯静态文件，不依赖任何框架和构建工具：
- **HTML5 + CSS3 + 原生 JavaScript**
- **localStorage / sessionStorage** — 数据存储
- **CSS 变量** — 主题切换
- **GitHub Pages** — 免费托管部署

## 如何当作博客模板使用

### 1. Fork/克隆项目

```bash
git clone https://github.com/JFClimbing/blog.git my-blog
cd my-blog
```

### 2. 修改基本信息

- 将 `index.html` 中的标题、描述改为你自己的
- 将 `about.html` 中的个人信息改为你自己的
- 修改导航栏链接为你的自定义页面

### 3. 修改管理密码

在以下3个文件中搜索 `LBS20040816`，替换为你自己的密码：
- `editor.html` — 文章编辑器密码
- `site-editor.html` — 网站编辑器密码  
- `js/admin.js` — 行内编辑模式密码

**注意：** 这是纯前端密码验证，不依赖服务器。安全性依赖于密码不泄露。

### 4. 自定义页面

每个页面是独立的 HTML 文件，可以随意增删：
```
├── index.html          # 首页 → 改这里
├── blog.html           # 文章列表
├── techniques.html     # 技术分享 → 新增的页面
├── yanxing.html        # 路线计划
├── about.html          # 关于页面
├── editor.html         # 写文章后台（密码保护）
├── site-editor.html    # 网站管理后台（密码保护）
├── css/
│   └── style.css       # 全局样式 → 改颜色、字体
├── js/
│   ├── main.js         # 核心逻辑 → 改文章数据
│   └── admin.js        # 后台编辑逻辑
└── posts/
    └── *.html          # 独立文章文件
```

### 5. 部署到 GitHub Pages

1. 将代码推送到你的 GitHub 仓库
2. 进入仓库 **Settings → Pages**
3. Source 选择 `main` 分支，根目录 `/`
4. 保存后等待1-2分钟，访问 `https://你的用户名.github.io/仓库名/`

### 6. 域名绑定（可选）

在仓库根目录创建 `CNAME` 文件，写入你的自定义域名，然后在 DNS 中添加 CNAME 记录指向 `你的用户名.github.io`。

## 如何发布文章

### 方法一：在线编辑器（推荐）
1. 在【关于】页面底部，点击 **写文章** 或 **网站编辑**
2. 输入密码进入后台
3. 撰写文章 → 生成 HTML → 复制代码
4. 在 GitHub 仓库的 `posts/` 目录创建新文件并粘贴
5. 在 `js/main.js` 的 `DEFAULT_POSTS` 数组中添加新文章信息

### 方法二：快捷行内编辑
1. 点击页面右下角 🔑 按钮，输入密码
2. 顶部出现编辑工具栏，可在页面上直接修改文字
3. 修改文章显隐状态、插入图片等

## 发布技术资料

1. 进入【技术分享】页面，滚动到底部
2. 点击 **上传技术资料**，输入博主密码
3. 填写标题、描述、选择分类
4. 上传 PDF 文件（建议不超过 2MB）或填写在线链接
5. 点击发布，文章即出现在列表顶部
6. 访客可以点击 **下载 PDF** 按钮直接下载

## 设计风格

- **暗色主题：** 深色背景 + 金色点缀，沉稳有质感
- **浅色主题：** 米白背景 + 棕色点缀，温暖舒适
- **字体：** 系统默认中文字体（苹方、微软雅黑）
- **卡片式布局：** 简洁清晰，信息层次分明

## 存储说明

所有数据存储在浏览器 localStorage/sessionStorage 中：

| 存储键 | 用途 |
|--------|------|
| `blog_posts` | 文章数据（覆盖默认文章列表） |
| `techniques_data` | 技术分享资料数据 |
| `site_data` | 全站配置（Hero文字、关于介绍等） |
| `inline_edits` | 行内编辑的修改内容 |
| `theme` | 主题偏好（dark/light） |
| `editor_auth` | 编辑器认证状态 |
| `site_editor_auth` | 网站编辑器认证状态 |
| `admin_auth` | 行内编辑模式认证状态 |
| `techniques_auth` | 技术资料上传认证状态 |

**注意：** localStorage 数据仅保存在你的浏览器中，不会自动同步到 GitHub。如需永久保存，请用网站编辑器的「发布更新」功能导出代码并更新 main.js。
